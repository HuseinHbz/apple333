## CI evidence contract

When the `Verify application quality` job reaches E2E, it now retains the
following `playwright-e2e-evidence` artifact for 14 days on **every** outcome:

| Evidence | Source | Retention behavior |
| --- | --- | --- |
| HTML report | `playwright-report/` | Generated by the CI-only HTML reporter. |
| Test output and attachments | `test-results/` | Explicitly preserved in CI, including existing axe JSON attachments. |
| Failure screenshots | `test-results/` | Captured with `screenshot: 'only-on-failure'`. |
| Failure traces | `test-results/` | Captured with `trace: 'retain-on-failure'`. |
| Failure videos | `test-results/` | Captured with `video: 'retain-on-failure'`. |
| Application and test-runner diagnostics | `logs/e2e/playwright-and-server.log` | Captured from the Playwright-managed server and test runner after sanitization. |

The Playwright web server has CI-only stdout piping enabled and stderr piping
enabled. The workflow routes the combined stream through
`scripts/sanitize-e2e-log.mjs`; no raw copy of that stream is persisted or
uploaded. The sanitizer redacts common credential-bearing URLs, bearer tokens,
cookies, sensitive environment values, and token-like query parameters. It does
not claim to inspect encrypted Playwright traces or video contents.
