# استراتژی UX/UI و Design System

## اصل تجربه

ظاهر باید مینیمال، آرام، دقیق و ممتاز باشد؛ تجربه باید به‌اندازهٔ Apple ساده، اما در نقطه‌های اعتماد (قیمت، موجودی، اقساط و Trade-in) صریح و قابل راستی‌آزمایی باشد. «لوکس» به معنی فضای خالی، تایپوگرافی دقیق، حرکت کم و محتوای شفاف است؛ نه تزئین اضافه.

## سیستم طراحی

| Token | مقدار پیشنهادی | کاربرد |
|---|---|---|
| Ink | `#0B0B0B` | متن/CTA اصلی |
| Paper | `#F5F5F2` | پس‌زمینهٔ گرم و ممتاز |
| White | `#FFFFFF` | سطح کارت و فرم |
| Gray 500 | `#737373` | متن ثانویه |
| Cyan accent | `#62D8F7` | وضعیت/تعامل محدود و premium |
| Success | `#16784A` | موجودی/تأیید |
| Warning | `#A66B15` | توجه مالی و موجودی |

- grid دسکتاپ: ۱۲ ستون، حداکثر 1440px، gutter 24px؛ نمایش 1920px با محتوای مرکزی و فضای تنفسی.
- موبایل: حداقل 320px، هدف iPhone، hit target حداقل 44px و bottom sheet برای انتخاب‌های پیچیده.
- type: Vazirmatn برای فارسی و یک فونت monospace محدود برای SKU/مقادیر عملیاتی؛ مقیاس 12/14/16/20/28/40/56.
- radius محدود 4 تا 12px؛ سایه کم؛ انیمیشن 160 تا 240ms و احترام به `prefers-reduced-motion`.

## ناوبری و صفحه‌ها

### فروشگاه

1. Header: لوگو، دسته‌ها، جست‌وجو، شعبه/مکان، حساب و سبد.
2. Home: Hero یک پیام، دسته‌ها، محصولات منتخب، Trade-in، اقساط، مقالات و اعتمادسازی.
3. PLP: فیلتر facet، sort، تعداد نتایج، موجودی شعبه و کارت محصول با قیمت/قسط.
4. PDP: گالری، انتخاب variant، قیمت و موجودی لحظه‌ای، CTA، اقساط، Trade-in، مشخصات، مقایسه، review/FAQ.
5. Checkout: اطلاعات تماس → تحویل → پرداخت → بازبینی؛ progress روشن و امکان بازگشت بدون از دست‌رفتن داده.

### عملیات داخلی

Sidebar ماژول‌ها، topbar با اعلان و اقدام اصلی، داشبورد role-aware و جدول‌های قابل فیلتر. عملیات مخرب نیازمند confirmation، علت و سطح دسترسی هستند. فیلدهای مالی/IMEI با ماسک و reveal کنترل‌شده نمایش می‌یابند.

## جریان‌های حساس

- **اقساط:** ابتدا estimate غیرتعهدآور، سپس مدارک، وضعیت بررسی و قرارداد. هر صفحه باید مرحله، مدارک لازم و زمان تقریبی را واضح بگوید.
- **Trade-in:** مدل/IMEI → خوداظهاری سلامت → ارزش تقریبی با بازه → نوبت ارزیابی → پیشنهاد نهایی؛ تفاوت estimate و پیشنهاد نهایی شفاف است.
- **Click & Collect:** انتخاب شعبه فقط پس از تایید موجودی؛ کد تحویل OTP و بازهٔ نگهداری رزرو به مشتری نمایش داده می‌شود.

## دسترس‌پذیری و کیفیت

- کنتراست AA، keyboard navigation، focus قابل مشاهده، label واقعی فرم و پیام خطای قابل فهم.
- skeleton در بارگذاری، empty/error state طراحی‌شده، نه صفحهٔ سفید.
- Content فارسی طبیعی، رقم و واحد پول یکدست، زمان نسبی همراه تاریخ دقیق در عملیات مالی.

## Quality Gate طراحی

هر flow پیش از توسعه با prototype قابل کلیک و تست ۵ کاربر هدف ارزیابی می‌شود. نمره‌ها: کیفیت بصری، UX، تبدیل، امکان‌پذیری فنی و حس premium. آستانهٔ انتشار **۹.۸/۱۰** است؛ نمرهٔ پایین‌تر نیازمند بازطراحی، ثبت یافته و ارزیابی مجدد است. این آستانه یک هدف کیفی است، نه جایگزین آزمون با کاربر یا معیارهای دسترس‌پذیری.
